# 기획 결손 진단 프롬프트 (MiniCut Timer 용)

아래 자료를 바탕으로 **기능 추가 제안이 아니라 기획 결손(전략/가설/지표/출시/리스크)만** 진단해줘.

## 입력 자료
- PRD 요약:
  - 문제: 2~6주 미니컷 사용자가 간단히 계획/기록/복기하고 싶다.
  - 목표: 계획 설정, 일일 기록, 달력 확인, 오프라인 저장, 플레이스토어 품질.
  - 비목표: 의료진단/클라우드/매크로/바코드/소셜.
- 현재 기능 요약:
  - 홈: 오늘 섭취/남음/초과, 빠른기록, 리마인더 설정, 주간 복기, 유지모드 체크
  - 플랜: 기간/목표 설정, 가이드 문구, 데이터 전체삭제
  - 캘린더: 일자별 기록 확인/삭제
  - 온보딩/한국어 카피/오프라인 로컬 저장
  - 광고 배너(AdMob) 노출
- 제약:
  - 지금은 기능을 더 넣기보다 **기획 완성도**를 높이고 싶다.

## 평가 프레임 (8개 축)
1. 문제-고객 적합성 (누구의 어떤 고통을 해결하는지 선명한가)
2. 가치제안/포지셔닝 (경쟁 대안 대비 왜 이 앱을 써야 하는가)
3. 성공지표 체계 (North Star + 입력지표 + 출시 합격선)
4. 사용자 여정/습관 설계 (온보딩→첫가치→7일 유지 루프)
5. 리스크/안전/신뢰 (오해 유발, 과도한 감량 유도, 법/정책)
6. 수익화/비즈니스 일관성 (광고 포함 시 UX/신뢰와 충돌 없는가)
7. 출시전략/GTM (스토어 실험, 리뷰수집, 카피 검증 계획)
8. 운영계획 (릴리즈 후 실험/개선 사이클)

## 출력 형식
- A. 8개 축 점수표 (0~5점, 근거 1줄)
- B. 상위 기획 결손 TOP 5
  - 결손명
  - 왜 중요한지
  - 현재 증거(있는 것/없는 것)
  - 1주 내 실행 가능한 보완 액션(문서/실험/운영 중심, 기능개발 최소화)
- C. "지금 당장 해야 할 3가지" (우선순위 순)
- D. 출시 가능 판정: Go / Conditional Go / No-Go + 조건

## 엄수 조건
- 새로운 기능 아이디어를 남발하지 말고, 먼저 기획 결손을 메우는 액션 중심으로 제안할 것.
- 모르면 추정이라고 명시할 것.
