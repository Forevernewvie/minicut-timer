# MiniCut 체류시간·재방문 기능 기획 프롬프트 실행 결과

작성일: 2026-04-28 (KST)
대상: MiniCut Android 앱
목표: 앱 체류시간을 무작정 늘리는 것이 아니라, 사용자가 미니컷을 더 잘 완주하도록 만드는 **의미 있는 체류시간·재방문 루프**를 설계한다.

---

## 1) 실행한 기획 프롬프트

```text
당신은 MiniCut Timer의 PM·프로덕트 디자이너·안드로이드 엔지니어로 구성된 Product Trio입니다.

제품 맥락:
- Android 전용 미니컷 앱.
- 현재 기능: 온보딩, 2~6주 플랜 생성, 하루 목표 칼로리 설정, 오늘 음식/칼로리 빠른 기록, 홈 대시보드, 달력 로그, 주간 복기, 오늘 코칭, 회복/근력/다이어트 브레이크 신호, 알림, AdMob/UMP.
- 포지셔닝: 장기 다이어트 앱이 아니라 짧고 선명한 2~6주 미니컷 코치.
- 제약: 사용자를 오래 붙잡기 위해 불필요한 콘텐츠 피드/중독형 UX를 넣지 않는다. 체류시간은 플랜 완주, 기록 지속, 회복 체크, 복기 품질을 높이는 방향이어야 한다.

기획 목표:
- 앱 체류시간과 재방문을 늘릴 기능을 제안하라.
- 단, 미니컷 도메인 특성상 사용자의 건강/신뢰를 해치거나 기록 피로를 늘리는 기능은 배제하라.
- 성공지표는 D1/D7 유지율, 일일 기록 완료율, 체크인 완료율, 플랜 완주율, 광고 노출 기회 증가와 함께 해석하라.

진행 방식:
1. PM 관점에서 5개 아이디어를 제시하라.
2. Designer 관점에서 5개 아이디어를 제시하라.
3. Engineer 관점에서 5개 아이디어를 제시하라.
4. 총 15개 중 Impact / Effort / Risk / Strategic Alignment 기준으로 상위 5개를 우선순위화하라.
5. 상위 5개 각각에 대해 다음을 작성하라:
   - 기능명
   - 한 줄 설명
   - 사용자가 앱에 머무는 이유
   - 기대 KPI
   - MVP 범위
   - 검증할 가정
   - 제외할 범위
6. 최종적으로 다음 릴리스에 넣을 P0/P1/P2 로드맵을 제안하라.
```

## 2) Product Trio 아이디어 발산

### PM 관점 5개

1. **오늘의 미니컷 미션**
   - 하루에 2~3개 핵심 행동(음식 1개 기록, 단백질 확인, 회복 체크)을 미션처럼 제시.
2. **3분 체크인 스트릭**
   - 연속 체크인 일수와 이번 플랜 내 체크인 완성도를 보여줌.
3. **주간 코칭 리포트**
   - 7일 기록을 바탕으로 “잘한 점 / 위험 신호 / 다음 주 한 가지”를 요약.
4. **플랜 완주 타임라인**
   - 2~6주 동안 현재 위치, 남은 일수, 전환 예정일을 시각화.
5. **브레이크/유지 전환 가이드**
   - 회복 위험이 누적될 때 3~7일 유지 전환 절차를 안내.

### Designer 관점 5개

1. **오늘 카드 중심 홈 리디자인**
   - 홈 첫 화면을 “오늘 해야 할 일” 중심으로 재구성.
2. **체크인 후 즉시 피드백 애니메이션**
   - 기록/체크인 완료 직후 짧은 긍정 피드백과 다음 행동 제시.
3. **캘린더 히트맵/리듬 뷰**
   - 날짜별 기록 강도, 목표 이내 여부, 체크인 여부를 한눈에 보여줌.
4. **문장형 코치 피드**
   - 숫자보다 해석 문장을 우선 노출해 사용자가 더 오래 읽고 이해하게 함.
5. **종료 D-day 시각화**
   - 남은 기간을 시각적으로 보여줘 앱을 다시 열 이유를 강화.

### Engineer 관점 5개

1. **로컬 인사이트 엔진**
   - Room 데이터를 집계해 오늘/주간/플랜 단위 인사이트를 오프라인 생성.
2. **미션 상태 머신**
   - 오늘 할 일을 규칙 기반으로 생성하고 완료 상태를 저장.
3. **이탈 위험 감지 규칙**
   - 2일 미기록, 과도한 초과, 회복 위험 누적 시 알림/홈 카드 변화.
4. **복기 스냅샷 저장**
   - 주간 리포트를 생성해 과거 리포트 아카이브로 남김.
5. **알림 개인화**
   - 사용자의 기록 시간대/미완료 상태에 따라 오전·저녁 메시지 조정.

## 4) 상위 5개 기능 상세 기획

### 1. 오늘의 미니컷 미션

- **한 줄 설명:** 홈 상단에 오늘 완료해야 할 2~3개 행동을 체크리스트로 보여준다.
- **사용자가 앱에 머무는 이유:** “오늘 무엇을 해야 하는지”가 명확하고, 각 항목을 완료하기 위해 기록/체크인/복기 화면을 자연스럽게 탐색한다.
- **기대 KPI:** D1 유지율, 일일 기록 완료율, 체크인 완료율, 세션당 화면 이동 수, 광고 노출 기회.
- **MVP 범위:**
  - 오늘 음식 1개 기록
  - 오늘 체크인 완료
  - 목표/남은 칼로리 확인
  - 완료 상태 홈 카드 표시
- **검증할 가정:** 사용자는 자유로운 대시보드보다 “오늘 해야 할 일”이 있을 때 더 자주 앱을 연다.
- **제외할 범위:** 포인트, 레벨, 경쟁 랭킹, 과도한 게임화.

### 2. 주간 코칭 리포트

- **한 줄 설명:** 최근 7일 데이터를 바탕으로 잘한 점, 위험 신호, 다음 주 한 가지 행동을 요약한다.
- **사용자가 앱에 머무는 이유:** 사용자는 자신의 기록이 해석되는 경험을 보기 위해 앱에 머문다.
- **기대 KPI:** D7 유지율, 주간 복기 진입률, 체크인 지속률, 플랜 완주율.
- **MVP 범위:**
  - 기록한 날 수
  - 목표 이내 일수
  - 평균 칼로리
  - 회복/근력/유지 신호 요약
  - 다음 주 제안 1개
- **검증할 가정:** 단순 달력보다 해석된 리포트가 재방문 이유를 더 강하게 만든다.
- **제외할 범위:** AI 서버 요약, 공유 이미지, 장문의 리포트.

### 3. 플랜 완주 타임라인 / D-day

- **한 줄 설명:** 현재 플랜의 시작일, 오늘 위치, 종료일, 남은 일수를 시각적으로 보여준다.
- **사용자가 앱에 머무는 이유:** 미니컷이 “끝이 있는 프로젝트”처럼 느껴져 매일 진행 상황을 확인하게 된다.
- **기대 KPI:** 플랜 완주율, D3/D7 유지율, 홈 체류시간, 플랜 탭 재방문율.
- **MVP 범위:**
  - 홈 카드에 D-day 표시
  - 진행률 바
  - 종료 후 리버스/유지 전환 예고 문구
- **검증할 가정:** 사용자는 남은 기간이 보일 때 중도 포기보다 완주 동기가 강해진다.
- **제외할 범위:** 복잡한 목표 그래프, 체중 예측 시뮬레이션.

### 4. 캘린더 히트맵 / 리듬 뷰

- **한 줄 설명:** 달력에 기록 여부, 목표 이내 여부, 체크인 완료 여부를 색/배지로 표시한다.
- **사용자가 앱에 머무는 이유:** 사용자는 자신의 패턴을 시각적으로 탐색하면서 과거 날짜를 눌러 복기한다.
- **기대 KPI:** 캘린더 탭 체류시간, 과거 기록 조회율, 기록 수정/삭제 정확도, D7 유지율.
- **MVP 범위:**
  - 기록 있음/없음
  - 목표 이내/초과
  - 체크인 완료 배지
  - 월간 요약 카드
- **검증할 가정:** 시각적 리듬이 있으면 사용자는 기록을 “모으는” 감각을 느낀다.
- **제외할 범위:** 소셜 공유, 상세 차트 라이브러리 추가.

### 5. 이탈 위험 감지 + 개인화 알림

- **한 줄 설명:** 2일 미기록, 연속 초과, 회복 위험 누적 등 이탈 신호를 감지해 알림/홈 메시지를 바꾼다.
- **사용자가 앱에 머무는 이유:** 앱이 사용자의 상태에 맞춰 필요한 시점에 다시 돌아오게 만든다.
- **기대 KPI:** D3/D7 유지율, 알림 클릭률, 미기록 복귀율, 체크인 재개율.
- **MVP 범위:**
  - 2일 미기록 복귀 메시지
  - 저녁 미완료 리마인더
  - 회복 위험 누적 시 “줄이기보다 회복 확인” 메시지
- **검증할 가정:** 일반 알림보다 상태 기반 알림이 복귀율을 높인다.
- **제외할 범위:** 푸시 A/B 플랫폼, 서버 기반 개인화, 과도한 알림 빈도.

## 5) 최종 권장 로드맵

### P0 — 다음 릴리스 후보

1. **오늘의 미니컷 미션**
2. **플랜 완주 타임라인 / D-day**
3. **주간 코칭 리포트 강화**

이 세 가지가 가장 좋다. 이유는 현재 앱 구조(Home/Calendar/Plan/Coach)를 크게 흔들지 않으면서, 사용자가 “오늘 다시 들어와야 하는 이유”를 만든다.

### P1 — P0 이후

4. **캘린더 히트맵 / 리듬 뷰**
5. **이탈 위험 감지 + 개인화 알림**

P1은 유지율에 강하지만 UX/알림 피로 리스크가 있어 P0 데이터 확인 후 넣는 것이 안전하다.

### P2 — 후속 확장

- 체크인 스트릭
- 복기 스냅샷 저장
- 브레이크/유지 전환 위저드
- 체크인 후 피드백 애니메이션

## 6) 최종 결론

체류시간을 늘리기 위해 가장 먼저 추가할 기능은 **콘텐츠 피드나 커뮤니티가 아니라 “오늘의 미니컷 미션”**이다.

MiniCut의 특성상 사용자가 오래 머물 이유는 재미보다 **오늘 기록하고, 체크하고, 해석받고, 완주에 가까워지는 느낌**이어야 한다. 따라서 다음 기능 묶음이 가장 적합하다.

1. 홈 상단: 오늘의 미니컷 미션
2. 홈/플랜: 플랜 D-day 진행률
3. 홈/캘린더: 주간 코칭 리포트 강화

이 조합은 앱 체류시간뿐 아니라 D1/D7 유지율, 일일 기록 완료율, 플랜 완주율까지 함께 개선할 가능성이 높다.
